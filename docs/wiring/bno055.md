# BNO055 Wiring

**Camera:** M5Stack Timer Camera F (U082-F)
**IMU breakout:** Adafruit BNO055 (RB-Ada-215 / Adafruit product #2472)

---

## Connection via Grove Port

The Timer Camera F exposes a Grove-compatible connector that carries 5 V power
and the ESP32's secondary I2C bus (Wire1, GPIO 4 / GPIO 13). The Adafruit
BNO055 breakout accepts 3.3–5 V on VIN, so it connects directly to the Grove
port with no level-shifting required.

> The camera's own internal I2C bus (Wire0, GPIO 25 SDA / GPIO 23 SCL) drives
> the OV3660 sensor and is separate — there is no bus conflict.

### Wiring Table

| Adafruit BNO055 pin | Grove / ESP32 pin | Notes                          |
|---------------------|-------------------|--------------------------------|
| VIN                 | Grove 5 V         | Onboard regulator → 3.3 V     |
| GND                 | Grove GND         |                                |
| SDA                 | GPIO 4 (Grove SDA) | `IMU_SDA_PIN` in `config.h`  |
| SCL                 | GPIO 13 (Grove SCL) | `IMU_SCL_PIN` in `config.h` |
| ADR                 | GND               | I2C address = 0x28            |
| INT                 | —                 | Not used                       |
| RST                 | —                 | Not used                       |

### Grove Connector Pin-Out (Timer Camera F)

```
┌───────────────────┐
│  1: GND           │
│  2: 5 V           │
│  3: GPIO 4  (SDA) │
│  4: GPIO 13 (SCL) │
└───────────────────┘
```

---

## I2C Address

| ADR pin  | I2C address | `config.h` value |
|----------|-------------|------------------|
| GND (default) | 0x28  | `#define BNO055_ADDR 0x28` |
| 3V3      | 0x29        | `#define BNO055_ADDR 0x29` |

Leave ADR unconnected or tie to GND for the default 0x28 address.

## Mounting Orientation

Mount the BNO055 breakout with:

- **X-axis** pointing forward (toward the camera lens direction)
- **Y-axis** pointing right
- **Z-axis** pointing up (same direction as the fisheye lens)

If you mount it in a different orientation, call `bno.setAxisRemap()` and
`bno.setAxisSign()` in `imu.cpp` to match your physical layout. Incorrect
orientation will produce wrong heading and tilt values.

## Calibration

The BNO055 self-calibrates continuously after power-on. Before the first
measurement session:

1. Power on — wait ~10 seconds for the gyroscope to settle.
2. **Magnetometer**: wave the device slowly in a figure-8 pattern until mag
   calibration reaches `3`.
3. **Accelerometer**: tilt the device to 6+ different stable orientations
   (each held for a few seconds) until accel calibration reaches `3`.
4. **Gyroscope**: leave the device perfectly still for a few seconds.

Target calibration: `sys=3 / gyro=3 / accel=3 / mag=3`.

Calibration offsets can be saved to ESP32 flash so the sensor reaches full
accuracy faster on subsequent power-ons — see the TODO for the implementation
task (`Adafruit_BNO055::getSensorOffsets` / `setSensorOffsets`).
