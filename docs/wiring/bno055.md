# BNO055 Wiring — M5Stack Timer Camera

The BNO055 is connected to the ESP32's **secondary I2C bus** (Wire1) to avoid
conflicts with the OV3660 camera's internal I2C bus.

## Pin Connections

| BNO055 Pin | ESP32 Pin | Notes                              |
|------------|-----------|------------------------------------|
| VIN        | 3V3       | 3.3 V supply                       |
| GND        | GND       |                                    |
| SDA        | GPIO 13   | `IMU_SDA_PIN` in `config.h`        |
| SCL        | GPIO 14   | `IMU_SCL_PIN` in `config.h`        |
| ADR        | GND       | I2C address = 0x28                 |
| INT        | —         | Not used                           |
| RST        | —         | Not used (software reset only)     |

> **ADR pin**: Pull ADR HIGH to change the I2C address to 0x29 and update
> `BNO055_ADDR` in `config.h` accordingly.

## I2C Address

| ADR Pin | Address |
|---------|---------|
| LOW (GND) | 0x28  |
| HIGH (3V3) | 0x29 |

## Mounting Orientation

The BNO055 fusion output assumes a specific axis orientation. Mount the sensor
with:
- **X-axis** pointing forward (toward the camera lens)
- **Y-axis** pointing right
- **Z-axis** pointing up

If the sensor is mounted differently, configure `Adafruit_BNO055::setAxisRemap()`
to match your physical orientation.

## Calibration

The BNO055 performs continuous self-calibration. For best accuracy before a
measurement session:
1. Power on and wait ~10 seconds.
2. Wave the sensor in a figure-8 pattern to calibrate the magnetometer.
3. Tilt the sensor to multiple orientations to calibrate the accelerometer.
4. A calibration status of `3/3/3/3` (sys/gyro/accel/mag) is ideal.

Calibration offsets can be stored in flash for faster startup — see
`Adafruit_BNO055::getSensorOffsets()`.
